# NeuroNote – GitLab Issue Plan

**Version:** 1.0.0  
**Last Updated:** 2026-06-28

This document defines 15 GitLab issues for the NeuroNote project. Each issue includes a title, detailed description, priority, estimated time, assignee (Member A or Member B), due date, and dependency.

| # | Issue Title | Description | Priority | Time | Assignee | Due | Depends On |
|---|---|---|---|---|---|---|---|
| 1 | **Repository Setup and Configuration** | Initialize the Git repository with README, SPEC, LICENSE, and .gitignore. Configure Python virtual environment. Create the folder structure (src/, models/, database/, data/, assets/, tests/, scripts/, examples/). Set up requirements.txt with all core dependencies (streamlit, PyMuPDF, pytesseract, numpy, opencv-python, networkx). | Critical | 2 hrs | Member A | Week 1, Day 1 | — |
| 2 | **Streamlit UI Scaffold and Dashboard** | Create the main Streamlit application entry point (src/app.py). Build the dashboard page displaying system statistics: total documents processed, concepts extracted, subjects available, and recent activity. Set up sidebar navigation with links to Upload, Search, Quiz, Knowledge Graph, and Dashboard pages. | Critical | 4 hrs | Member B | Week 1, Day 2 | Issue 1 |
| 3 | **File Upload Interface** | Build the file upload page supporting drag-and-drop and browse for PDF, image (JPEG/PNG/TIFF), audio (MP3/WAV/M4A), and text files. Add metadata input fields (subject name, tags). Implement file validation (type check, size limits). Show processing progress bars. | Critical | 4 hrs | Member B | Week 1, Day 3 | Issue 2 |
| 4 | **OCR Integration with Tesseract** | Integrate Tesseract OCR via pytesseract. Implement image preprocessing pipeline: grayscale conversion, binarization (OTSU threshold), denoising, deskewing, and scaling to 300 DPI. Support printed text, handwritten notes, and whiteboard photos with appropriate preprocessing settings. Store OCR output text in the database. | Critical | 4 hrs | Member A | Week 1, Day 3 | Issue 1 |
| 5 | **Whisper.cpp Speech-to-Text Integration** | Integrate Whisper.cpp for offline audio transcription. Implement audio format conversion to 16-bit WAV at 16 kHz. Segment long audio into 30-second chunks for processing. Use the ggml-small quantized model for CPU-optimized performance. Store transcribed text in the database with metadata. | Critical | 4 hrs | Member A | Week 1, Day 4 | Issue 1 |
| 6 | **SQLite Database Schema and Initialization** | Design and implement the SQLite database schema with 10 tables: documents, subjects, topics, definitions, questions, audio, images, knowledge_links, quiz_history, revision_history. Enable WAL journal mode for concurrent read/write. Create FTS5 virtual table for full-text search. Implement connection manager with context manager support. | Critical | 3 hrs | Member A | Week 2, Day 1 | Issue 1 |
| 7 | **LLM Integration with llama.cpp and Phi-3 Mini** | Integrate llama.cpp Python bindings. Implement lazy model loading of Phi-3 Mini GGUF (Q4_K_M) from models/ directory. Implement memory cleanup (unload model after idle timeout). | Critical | 5 hrs | Member A | Week 2, Day 2 | Issue 1 |
| 8 | **PDF Processing with PyMuPDF** | Implement PDF text extraction using PyMuPDF (fitz). Extract text from all pages while preserving document structure (headings, paragraphs). Extract embedded images from PDF pages. Handle multi-page documents (up to 100 pages). Implement error handling for encrypted or corrupted PDFs. Store extracted text in the documents table. | High | 3 hrs | Member A | Week 2, Day 1 | Issue 1 |
| 9 | **Concept Extraction and JSON Structuring** | Build the LLM-based concept extraction pipeline. Design prompts for concept extraction, definition identification, topic classification, and relationship detection. Implement JSON parsing and validation of LLM output. Aggregate concepts across chunks. Deduplicate identical terms. Store results in definitions and topics tables. | High | 5 hrs | Member A | Week 2, Day 3 | Issue 7 |
| 10 | **Knowledge Graph Construction and Visualization** | Build the knowledge graph using NetworkX. Create graph nodes from extracted concepts. Compute TF-IDF similarity between concept definitions. Create relationship edges with 8 types (related_to, is_a, part_of, prerequisite, example_of, contrasts_with, causes, used_in). Serialize graph edges to the knowledge_links table. Build interactive visualization using Plotly or Pyvis with zoom, pan, and click-to-select functionality. | High | 5 hrs | Member A (graph) / Member B (viz) | Week 2, Day 4 | Issue 9 (A), Issue 2 (B) |
| 11 | **Full-Text Search Implementation** | Implement full-text search using SQLite FTS5. Build a search interface with text input field, autocomplete suggestions, and result display. Implement filters by subject, document type, and date range. Display ranked results with highlighted context snippets. Support basic query syntax (AND, OR, phrase matching). Ensure response time under 2 seconds. | High | 3 hrs | Member B | Week 2, Day 3 | Issue 6, Issue 2 |
| 12 | **Quiz Generator Interface** | Build the quiz generation interface. Allow user to select subject, topic, number of questions (5/10/15/20), and difficulty level. Use the LLM to generate multiple-choice, true/false, and short-answer questions. Display questions one at a time with progress indicator. Implement answer submission, scoring, and results display. Store quiz attempts in quiz_history table. | Medium | 4 hrs | Member B | Week 2, Day 4 | Issue 7, Issue 2 |
| 13 | **Revision Planner** | Build the revision planner interface. Allow user to select a subject and specify available study duration (days and hours per day). Algorithm prioritizes topics by concept count and importance. Generate a day-by-day study schedule with topic names, concept counts, and practice question counts. Support export as Markdown or JSON. | Medium | 3 hrs | Member B | Week 3, Day 1 | Issue 9, Issue 2 |
| 14 | **Unit and Integration Testing** | Write unit tests for all processing modules: PDF extraction, OCR, text preprocessing, LLM inference, concept extraction, and database operations. Write integration tests for end-to-end workflows: PDF upload through search, image upload through concept extraction, and text input through quiz generation. Achieve >80% code coverage. | High | 5 hrs | Member A | Week 3, Day 2 | Issues 4, 5, 7, 8, 9 |
| 15 | **Documentation and Demo Preparation** | Finalize all documentation. Update README with setup instructions and screenshots. Prepare submission package. | High | 4 hrs | Member B | Week 3, Day 3 | All issues |
